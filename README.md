# Εφαρμογή Καταγραφής Ασθενών Νοσοκομείου

Μικρή web εφαρμογή με PHP και MySQL για καταγραφή, εμφάνιση, αναζήτηση, επεξεργασία και διαγραφή ασθενών νοσοκομείου.

## Λειτουργίες

- Καταχώρηση ασθενή με όνομα, επώνυμο, ΑΜΚΑ, ύψος, βάρος, ημερομηνία γέννησης, ημερομηνία εισαγωγής και αιτία εισαγωγής.
- Αποθήκευση των στοιχείων σε βάση δεδομένων MySQL.
- Προβολή όλων των ασθενών σε πίνακα.
- Αναζήτηση με όνομα, επώνυμο ή ΑΜΚΑ.
- Επεξεργασία στοιχείων ασθενή.
- Διαγραφή ασθενή με επιβεβαίωση.
- Βασικοί έλεγχοι εγκυρότητας για υποχρεωτικά πεδία, ΑΜΚΑ, αριθμητικά πεδία και ημερομηνίες.

## Διαδικασία Ανάπτυξης της Εφαρμογής

Η ανάπτυξη της εφαρμογής έγινε σταδιακά με τα παρακάτω βήματα:

1. Καθορίστηκαν τα δεδομένα που πρέπει να αποθηκεύονται για κάθε ασθενή.
2. Σχεδιάστηκε η βάση δεδομένων MySQL και δημιουργήθηκε ο πίνακας `patients`.
3. Δημιουργήθηκε το αρχείο `db.php` για τη σύνδεση της εφαρμογής με τη βάση δεδομένων.
4. Υλοποιήθηκε η σελίδα `index.php`, η οποία περιέχει τη φόρμα καταχώρησης ασθενή.
5. Δημιουργήθηκε το `save_patient.php`, το οποίο ελέγχει τα στοιχεία της φόρμας και τα αποθηκεύει στη βάση.
6. Υλοποιήθηκε το `patient_list.php`, όπου εμφανίζονται οι καταχωρημένοι ασθενείς και υπάρχει δυνατότητα αναζήτησης.
7. Προστέθηκε δυνατότητα επεξεργασίας ασθενή μέσω των αρχείων `edit_patient.php` και `update_patient.php`.
8. Προστέθηκε δυνατότητα διαγραφής ασθενή μέσω του `delete_patient.php`.
9. Δημιουργήθηκε το `style.css` για την εμφάνιση της εφαρμογής.
10. Έγιναν δοκιμές εισαγωγής, αναζήτησης, επεξεργασίας και διαγραφής εγγραφών.
11. Δημιουργήθηκε export της βάσης δεδομένων και αναρτήθηκε ο πηγαίος κώδικας σε δημόσιο GitHub repository.
    
## Αρχεία εφαρμογής

- `index.php`: φόρμα νέας καταχώρησης ασθενή.
- `save_patient.php`: αποθήκευση νέου ασθενή στη βάση.
- `patient_list.php`: λίστα, αναζήτηση και ενέργειες ασθενών.
- `edit_patient.php`: φόρμα επεξεργασίας ασθενή.
- `update_patient.php`: ενημέρωση στοιχείων ασθενή.
- `delete_patient.php`: διαγραφή ασθενή.
- `db.php`: σύνδεση με τη βάση MySQL.
- `patients.sql`: δημιουργία βάσης και πίνακα.
- `style.css`: εμφάνιση της εφαρμογής.
- `Δίκτυο(1).pkt`: δικτυακή τοπολογία στο Cisco Packet Tracer

## Βάση δεδομένων

Η εφαρμογή χρησιμοποιεί βάση δεδομένων MySQL με όνομα `hospital_db`. Ο βασικός πίνακας είναι ο `patients`, στον οποίο αποθηκεύονται τα στοιχεία των ασθενών.

Ο πίνακας `patients` περιλαμβάνει τα παρακάτω πεδία:

- `id`: μοναδικός αριθμός ασθενή, πρωτεύον κλειδί και auto increment.
- `first_name`: όνομα ασθενή.
- `last_name`: επώνυμο ασθενή.
- `amka`: ΑΜΚΑ ασθενή, μοναδικό πεδίο.
- `height`: ύψος ασθενή σε εκατοστά.
- `weight`: βάρος ασθενή σε κιλά.
- `birth_date`: ημερομηνία γέννησης.
- `admission_date`: ημερομηνία εισαγωγής στο νοσοκομείο.
- `reason`: αιτία εισαγωγής.
- `created_at`: ημερομηνία και ώρα δημιουργίας της εγγραφής.

Η βάση δημιουργείται από το αρχείο `patients.sql`, ενώ το αρχείο `hospital_db_export.sql` περιέχει export της βάσης δεδομένων.

Για εγκατάσταση:

1. Ανοίξτε το XAMPP.
2. Εκκινήστε Apache και MySQL.
3. Ανοίξτε το phpMyAdmin.
4. Κάντε import το αρχείο `patients.sql`.

## Εκτέλεση

Με PHP server:

```powershell
C:\xampp\php\php.exe -S 127.0.0.1:8000 -t .
```

Μετά ανοίξτε στον browser:

```text
http://127.0.0.1:8000/index.php
```

## Στοιχεία σύνδεσης

Οι ρυθμίσεις σύνδεσης βρίσκονται στο `db.php`:

```php
$host = 'localhost';
$database = 'hospital_db';
$username = 'root';
$password = '';
```

## Δικτυακή Αρχιτεκτονική

Για τη δικτυακή αρχιτεκτονική σχεδιάστηκε τοπολογία στο Cisco Packet Tracer. Η τοπολογία περιλαμβάνει υπολογιστές-πελάτες, δικτυακές συσκευές και server που φιλοξενεί την web εφαρμογή.

Η βασική λογική της αρχιτεκτονικής είναι client-server:

- Οι υπολογιστές-πελάτες χρησιμοποιούν web browser για πρόσβαση στην εφαρμογή.
- Ο server φιλοξενεί τα αρχεία της εφαρμογής και εξυπηρετεί HTTP αιτήματα.
- Η επικοινωνία γίνεται μέσω IP δικτύου.
- Κάθε συσκευή έχει κατάλληλη IP διεύθυνση, subnet mask και default gateway.
- Η συνδεσιμότητα ελέγχθηκε με εντολές όπως `ping` και `tracert`.

Παράδειγμα πρόσβασης στην εφαρμογή από client: http://128.1.2.3/index.html
